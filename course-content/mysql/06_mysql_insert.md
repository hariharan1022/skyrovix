## 6. MySQL Insert Data

## 📘 Introduction
`INSERT INTO` adds rows to a table. MySQL supports inserting single rows, multiple rows in one statement, conditional inserts with `INSERT IGNORE`, and upsert behavior with `ON DUPLICATE KEY UPDATE`. Understanding auto-increment behavior and retrieving the last inserted ID is essential.

## 🧠 Key Concepts
- **INSERT INTO ... VALUES**: Standard syntax for adding rows
- **Multiple Rows**: `VALUES (row1), (row2), (row3)` in a single INSERT
- **INSERT IGNORE**: Silently skips rows that cause duplicate-key or other errors
- **ON DUPLICATE KEY UPDATE**: If a duplicate key is found, update instead of insert (upsert)
- **AUTO_INCREMENT**: MySQL automatically assigns the next integer if omitted
- **LAST_INSERT_ID()**: Returns the first auto-generated ID from the most recent INSERT

## 💻 Syntax
```sql
-- Insert single row
INSERT INTO table_name (col1, col2, col3)
VALUES (val1, val2, val3);

-- Insert multiple rows
INSERT INTO table_name (col1, col2)
VALUES (val1, val2),
       (val3, val4),
       (val5, val6);

-- Insert with IGNORE (skip errors)
INSERT IGNORE INTO table_name (col1, col2)
VALUES (val1, val2);

-- Upsert (insert or update)
INSERT INTO table_name (col1, col2)
VALUES (val1, val2)
ON DUPLICATE KEY UPDATE col2 = VALUES(col2);
```

## ✅ Example 1 - Basic

**Problem**: Insert two students into the `students` table and retrieve the auto-generated IDs.

**SQL Query**:
```sql
INSERT INTO students (name, email, age)
VALUES ('Alice Johnson', 'alice@example.com', 20);

SELECT LAST_INSERT_ID() AS 'New ID';

INSERT INTO students (name, email)
VALUES ('Bob Smith', 'bob@example.com');

SELECT LAST_INSERT_ID() AS 'New ID';
```

**Output**:
```
+--------+
| New ID |
+--------+
|      1 |
+--------+

+--------+
| New ID |
+--------+
|      2 |
+--------+
```

**Explanation**: The first insert provides all columns (age = 20). The second omits `age` (defaults to 18) and `enrolled_at` (defaults to current date). Each `LAST_INSERT_ID()` returns the ID generated by the immediately preceding INSERT.

## 🚀 Example 2 - Intermediate

**Problem**: Insert multiple customers, then use `INSERT ... ON DUPLICATE KEY UPDATE` to upsert when attempting to re-insert an existing email.

**SQL Query**:
```sql
-- Create and populate customers
CREATE TABLE customers (
    id INT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(255) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    visits INT DEFAULT 1
);

INSERT INTO customers (email, name)
VALUES ('alice@example.com', 'Alice'),
       ('bob@example.com', 'Bob'),
       ('carol@example.com', 'Carol');

-- Upsert: increment visit count if email exists
INSERT INTO customers (email, name, visits)
VALUES ('alice@example.com', 'Alice', 1)
ON DUPLICATE KEY UPDATE visits = visits + 1;

SELECT * FROM customers;
```

**Output**:
```
+----+-------------------+-------+--------+
| id | email             | name  | visits |
+----+-------------------+-------+--------+
|  1 | alice@example.com | Alice |      2 |
|  2 | bob@example.com   | Bob   |      1 |
|  3 | carol@example.com | Carol |      1 |
+----+-------------------+-------+--------+
```

**Explanation**: The third statement tries to insert Alice again. Because `email` is UNIQUE, the duplicate key triggers the `UPDATE` clause — `visits` increments from 1 to 2. The rest of the row remains unchanged.

## 🏢 Real World Use Case
A logging system inserts thousands of events per second. Using multi-row INSERT (`INSERT INTO logs (event, ts) VALUES ('login', NOW()), ('logout', NOW()) ...`) reduces round-trips. `INSERT IGNORE` handles deduplication on idempotent retries without failing.

## 🎯 Interview Questions
1. **Q**: What does `LAST_INSERT_ID()` return?  
   **A**: The first auto-increment value generated by the most recent INSERT statement in the current session.
2. **Q**: What is the difference between `INSERT IGNORE` and `ON DUPLICATE KEY UPDATE`?  
   **A**: `INSERT IGNORE` silently skips rows that cause errors (including duplicates). `ON DUPLICATE KEY UPDATE` performs an update when a duplicate key is encountered.
3. **Q**: Can you insert multiple rows in a single INSERT?  
   **A**: Yes — `INSERT INTO t (col) VALUES (1), (2), (3);`
4. **Q**: What happens if you omit the auto-increment column in an INSERT?  
   **A**: MySQL automatically assigns the next sequential integer.
5. **Q**: How does `ON DUPLICATE KEY UPDATE` affect `LAST_INSERT_ID()`?  
   **A**: If an UPDATE occurs (not an INSERT), `LAST_INSERT_ID()` returns 0 unless `LAST_INSERT_ID(expr)` is used in the UPDATE clause.

## ⚠ Common Errors / Mistakes
- Inserting a NULL into a `NOT NULL` column
- Violating `UNIQUE` or `PRIMARY KEY` constraints (unless using `IGNORE` or `ON DUPLICATE KEY`)
- String values not quoted — MySQL treats unquoted tokens as column names
- Date values in wrong format (use `'YYYY-MM-DD'`)
- Forgetting column list order — relying on column order is fragile
