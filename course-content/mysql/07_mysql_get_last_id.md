## 7. MySQL Get Last ID

## 📘 Introduction
After inserting a row with `AUTO_INCREMENT`, you often need the generated ID to use in subsequent operations (e.g., inserting related rows). MySQL provides `LAST_INSERT_ID()` for this purpose. Different programming interfaces have their own methods (PHP's `mysqli_insert_id`, PDO's `lastInsertId`).

## 🧠 Key Concepts
- **LAST_INSERT_ID()**: Session-scoped function returning the first auto-generated ID from the last INSERT
- **Session Isolation**: `LAST_INSERT_ID()` is per-connection — concurrent inserts by other users do not affect it
- **@@identity**: A session variable in some SQL databases; MySQL supports it but `LAST_INSERT_ID()` is preferred
- **PHP mysqli_insert_id()**: Returns last ID from MySQLi procedural/OOP connection
- **PDO lastInsertId()**: Returns last ID from PDO connection
- **Multiple Rows**: `LAST_INSERT_ID()` returns the first ID generated; subsequent IDs are sequential

## 💻 Syntax
```sql
-- Get last inserted ID
SELECT LAST_INSERT_ID();

-- Using in statements
INSERT INTO users (name) VALUES ('Alice');
SET @new_user_id = LAST_INSERT_ID();
INSERT INTO profiles (user_id, bio) VALUES (@new_user_id, 'Hello!');

-- With multiple rows (returns first ID)
INSERT INTO users (name) VALUES ('Bob'), ('Carol'), ('Dave');
SELECT LAST_INSERT_ID(); -- Returns Bob's ID
```

## ✅ Example 1 - Basic

**Problem**: Insert a new customer, retrieve the new ID, and use it to create a related order.

**SQL Query**:
```sql
INSERT INTO customers (name, email)
VALUES ('Diana Prince', 'diana@example.com');

SET @customer_id = LAST_INSERT_ID();

INSERT INTO orders (customer_id, total, status)
VALUES (@customer_id, 299.99, 'pending');

SELECT @customer_id AS 'New Customer ID',
       (SELECT COUNT(*) FROM orders WHERE customer_id = @customer_id) AS 'Orders Created';
```

**Output**:
```
+-----------------+----------------+
| New Customer ID | Orders Created |
+-----------------+----------------+
|               4 |              1 |
+-----------------+----------------+
```

**Explanation**: `LAST_INSERT_ID()` captures Diana's auto-increment ID (assumed 4). The value is stored in a user variable `@customer_id` and reused to create the order. The subquery confirms the order was linked correctly.

## 🚀 Example 2 - Intermediate

**Problem**: Insert multiple rows and demonstrate that `LAST_INSERT_ID()` returns only the first ID, not the last.

**SQL Query**:
```sql
INSERT INTO customers (name, email)
VALUES ('Eve Adams', 'eve@example.com'),
       ('Frank Lee', 'frank@example.com'),
       ('Grace Kim', 'grace@example.com');

SELECT LAST_INSERT_ID() AS 'First ID of Batch',
       CONCAT('IDs ', LAST_INSERT_ID(), ' through ', LAST_INSERT_ID() + 2) AS 'ID Range';
```

**Output**:
```
+-----------------+----------------------+
| First ID of Batch | ID Range           |
+-----------------+----------------------+
|               5 | IDs 5 through 7      |
+-----------------+----------------------+
```

**Explanation**: The multi-row insert creates three customers. `LAST_INSERT_ID()` returns 5 (the first ID). The IDs 5, 6, 7 are assigned sequentially. This is important when inserting parent and child records — you know the range of IDs generated.

## 🏢 Real World Use Case
In an e-commerce checkout flow: `INSERT INTO orders ...` gets its ID via `LAST_INSERT_ID()`, then that ID is used for `INSERT INTO order_items (order_id, product_id, qty) VALUES (...)`. This guarantees the order and its line items are linked without race conditions.

## 🎯 Interview Questions
1. **Q**: Is `LAST_INSERT_ID()` safe in a multi-user environment?  
   **A**: Yes — it is session-scoped. Each connection only sees its own last inserted ID.
2. **Q**: What does `LAST_INSERT_ID()` return after a multi-row INSERT?  
   **A**: The first auto-increment value generated by that INSERT, not the last.
3. **Q**: What does `LAST_INSERT_ID()` return if no INSERT has been executed?  
   **A**: 0.
4. **Q**: How do you get the last inserted ID in PHP with PDO?  
   **A**: `$pdo->lastInsertId();`
5. **Q**: What happens to `LAST_INSERT_ID()` after an `INSERT ... ON DUPLICATE KEY UPDATE` where an UPDATE occurs?  
   **A**: It returns 0 unless you use `LAST_INSERT_ID(expr)` in the UPDATE clause.

## ⚠ Common Errors / Mistakes
- Calling `LAST_INSERT_ID()` in a different session (e.g., after reconnecting) — returns 0
- Assuming `LAST_INSERT_ID()` returns the last inserted ID of any table (it's per-statement)
- Using `SELECT MAX(id)` instead of `LAST_INSERT_ID()` — not safe under concurrent writes
- Forgetting to capture the ID immediately after INSERT before any other statements
